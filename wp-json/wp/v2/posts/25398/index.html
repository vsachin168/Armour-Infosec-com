\u201cWhat is Cronjob ?\u201d<\/strong><\/h5>\n

Cron<\/strong> is a utility that allows Linux users to do specific task on the server at a given time and date.Consider system admin is required to take the backup of server at regular interval. It means system admin has to do repeated task at certain interval.This repeated task at certain interval can be automated in Linux using cron utility. This automated repeated task is known as cronjob and a table or file that maintain this cronjob is known as crontab. Linux maintains separate crontab for each and every user.Given below is the syntax to define a cronjob in crontab<\/strong>.<\/p>\n

<\/p>\n

A single line in the crontab represent a cronjob. Cronjob is divided into three parts. All three parts are described below.<\/p>\n

When to perform cronjob ?<\/h5>\n

First five numeric value represents the time of execution of the cronjob. Now let’s understand the five numeric value.<\/p>\n

    \n
  • Minute\u00a0 –\u00a0 First value represents minute ranges between 0 to 59 and * means any minute.<\/li>\n
  • Hour\u00a0\u00a0\u00a0\u00a0\u00a0 –\u00a0Second value represent Hour ranges between 0 to 24 and * means any hour.<\/li>\n
  • Day of month\u00a0 –\u00a0Third value represents day of month ranges between 1 to 31 and * means any day.<\/li>\n
  • Month\u00a0 –\u00a0Fourth value represents month ranges between 1 to 12 and * means any month.<\/li>\n
  • Day of week\u00a0 –\u00a0Fifth value represents the day of week ranges between 0 to 6 starting from Sunday and * means any day of week.<\/li>\n<\/ul>\n
    By whom privileges does the task perform ?<\/h5>\n

    The value Just after the numeric value represents the user whose privileges will be used to accomplish the task.<\/p>\n

    Which command to be execute ?<\/h5>\n

    After defining the user we need to provide the command to be executed at that time.
    \nI hope we found our answer and now we will learn to escalate privileges through cronjob. For better understanding i am dividing further blog into two parts Enumeration and Exploitation.<\/p>\n

    Cronjob Enumeration<\/h4>\n

    Lets start with the enumeration of the cronjob. The cronjob enumeration includes, finding and understanding the task that cronjob was assinged.There are following types of cronjob that we have to find.<\/p>\n

    User based Cronjob<\/h5>\n

    In Linux each and every user can perform cronjobs. Each and every user maintains a crontab for their cronjobs. The location of the crontab of each user is in the following directory.<\/p>\n

    \/var\/spool\/cron\/crontabs\/'crontab_of_the_each_user_named_as_their_username'<\/pre>\n
    The above directory is only accessible through root user. Normal user can check their cronjobs using command.<\/p>\n
    crontab -l<\/pre>\n
    Application based Cronjob<\/h5>\n
    Certain application in Linux uses cronjob to perform their task. All the cronjobs that are created by any application is placed in the following directory.<\/p>\n
    \/etc\/cron.d<\/pre>\n
    Anacron<\/h5>\n
    Anacron is defined as the cron with ability to performed the task that are skipped due to some reasons.This type of cronjob are placed in the following directory.<\/p>\n
    \/etc\/anacrontab<\/pre>\n
    Above three are the possible ways to find the cronjobs. Now for understanding them only thing you have to do is read them.<\/p>\n
    Pro tip : If you want to know about the cronjobs of the other user then you can use the tool pspy(pspy32 for 32 bit and pspy64 for 64bit).<\/strong><\/p>\n
    Download link : https:\/\/github.com\/DominicBreuker\/pspy<\/a><\/p>\n
    Cronjob Exploitation<\/h4>\n
    From the above phase you found a cronjob and understands the working of the same.Now there are two possible ways to exploit the cronjob. There may be more!!!<\/p>\n
    For following demonstration I am using CentOS<\/strong> as target of evaluation and Kali Linux<\/strong> as attacker system.<\/p>\n
    File Overwrite<\/h5>\n
    Situation where the vulnerability arises :<\/strong><\/p>\n
    When a user defines a cronjob that runs a script, that script is writable by attacker then attacker can go for the file overwrite. Now consider a situation where sysadmin defines a cronjob that clears all files in \/tmp\/demo directory every 2 min. We will setup lab for the same situation and will exploit it.<\/p>\n
    Setting up the lab with same vulnerability :<\/strong><\/p>\n
    I had created a demo directory in tmp<\/strong> and placed some files in it (demo).<\/p>\n
    pwd\r\ntouch 1.txt 2.txt 3.txt 4.php\r\nls<\/pre>\n
    <\/p>\n
    Now lets create a python script to delete all files from demo and setting permissions to the file.<\/p>\n
    cat cleanup.py\r\nchmod 777 cleanup.py<\/pre>\n
    <\/p>\n
    Next thing we need to do is setup cronjob.<\/p>\n
    cat \/etc\/crontab<\/pre>\n
    <\/p>\n
    Now lets verify our cronjob.<\/p>\n
    ls\r\ndate\r\nls<\/pre>\n
    <\/p>\n
    Exploiting the vulnerability :<\/h5>\n
    We successfully created a cronjob and verified it now lets exploit it. On enumerating the target we see that there is a cronjob running on the system which running the script.<\/p>\n
    <\/p>\n
    Script is world writable file (777 permission) and we can edit it. There are many way from here to escalate privileges. We are going to set suid<\/strong> bit on \/bin\/bash<\/strong> by replacing \u201crm -r \/tmp\/demo<\/strong>\u201d from\u00a0 \u201cchmod u+s \/bin\/bash<\/strong>\u201d.<\/p>\n
    cat cleanup.py<\/pre>\n
    <\/p>\n
    After waiting two minute we will run \/bin\/bash to escalate privilege.<\/strong><\/p>\n
    whoami\r\n\/bin\/bash -p\r\nwhoami<\/pre>\n
    <\/p>\n
    Wildcard Injection :<\/h4>\n
    Situation where the vulnerability arises :<\/h5>\n
    When the command is\u00a0 assigned to a cronjob, contains a wildcard operator then attacker can go for wildcard injection to escalate privilege. Now consider a situation where sysadmin sets up a cronjob that creates a tar file, of all the content of user and store them in \/var\/backup<\/strong>.<\/p>\n
    Setting up the lab with same vulnerability:<\/h5>\n
    Lets create some file in \/home\/armour\/<\/strong> data directory which we will backup.<\/p>\n
    touch 1.txt 2.txtx 4.txt 5.php\r\nls<\/pre>\n
    <\/p>\n
    Now we schedule a cronjob to store the backup into \/var\/backup<\/strong><\/p>\n
    cat \/etc\/crontab<\/pre>\n
    <\/p>\n
    Lets verify out objective.<\/p>\n
    ls\r\ndate<\/pre>\n
